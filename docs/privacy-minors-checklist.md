# 未成年人数据与隐私操作清单

本清单用于小规模家教试点的实际操作控制，不替代专业法律意见。学生是否已满 14 周岁必须逐人确认；不论年龄，平台默认取得并记录监护人知情同意。处理不满 14 周岁未成年人个人信息时，应按敏感个人信息采取更严格保护并取得监护人同意。

## 试点前

- [ ] 向学生和监护人说明收集内容：账号资料、作业图片/PDF、错题、用时、自评、留言、AI 对话、复习记录和已发布周报。
- [ ] 说明用途：教学反馈、错因复核、复习安排、个性化答疑和备课；不得用于广告、公开展示或训练自有模型。
- [ ] 列明数据处理方与可能的数据出境情况：GitHub Pages、Supabase、AI/Embedding 供应商及其所在区域。
- [ ] 告知 AI 可能出错，所有正式学情结论和家长周报须由教师确认。
- [ ] 如为持续跟踪学情而长期保留原始附件、对话、错题和周报，应明确保留目的、数据范围和起始时间，并取得学生及监护人知情同意。
- [ ] 至少每年复核长期保留的必要性、范围和供应商副本；不再需要的数据按约定删除或匿名化，并始终保留撤回同意、导出和申请删除的路径。长期保留不等于永久且无需复核。
- [ ] 告知查阅、更正、导出、撤回同意和删除的联系渠道与预计处理时间。
- [ ] 记录学生年龄状态、监护人、同意版本、同意时间和获取方式；没有记录不得开通真实账号。
- [ ] 对供应商的数据保留、训练开关、日志、区域和安全条款做上线时复核，关闭允许用于模型训练的选项（如供应商提供）。

## 账号与权限

- [ ] 教师创建账号并通过私下渠道交付至少 8 位的临时密码；首次登录出现不可跳过的改密窗口，并要求设置至少 10 位且不与其他网站共用的新密码。
- [ ] 分别验证改密成功、Supabase 更新失败、后端确认失败和刷新重登：只有密码更新及 `acknowledge_password_change` 都成功后才能解除 `must_change_password` 标志。
- [ ] 每位家长只关联自己的学生；关系变更后立即撤销旧权限。
- [ ] 学生只能访问本人数据，家长只能访问已发布周报，教师特权操作留审计记录。
- [ ] 定期使用两个学生账号做交叉越权测试，包括猜测 URL、附件路径、搜索、AI 引用和导出接口。
- [ ] 不共用教师账号；教师设备启用锁屏、磁盘加密和受支持的浏览器。
- [ ] 不在 URL、文件名、日志或公开 issue 中写学生真实姓名、电话和其他可识别信息。

## 上传与 AI

- [ ] 上传前提示只拍题目和作答区域，裁去无关人脸、同学姓名、学校班级、二维码、地址和联系方式。
- [ ] 默认使用私有 Storage；附件访问通过 RLS 或短时签名 URL，不提供永久公开链接。
- [ ] 前端校验类型和大小，后端再次校验；拒绝可执行文件和与学习无关的内容。
- [ ] 向模型只发送完成当前任务所需的最少片段，不发送整份学生档案或其他学生资料。
- [ ] 讲义检索按学生和科目授权；`teacher_only` 永不进入学生答疑，`solution_gated` 仅在完整解答阶段使用。
- [ ] 学生上传内容视为不可信输入，不能让其中的提示词修改档案、调用管理功能或读取其他学生信息。
- [ ] AI 推测保持 `ai_inferred`，未经教师确认不得进入长期学情、复习结论或家长周报。
- [ ] 学习习惯只使用可追溯行为证据，不生成性格、心理健康、智力或家庭状况等无依据判断。

## 教师日常操作

- [ ] 每条正式错题记录保留原始提交、题号、证据类型、确认人和确认时间。
- [ ] 发布周报前逐项检查事实、措辞、对象和可见范围，避免把教师私密备注或 AI 草稿发给家长。
- [ ] 发送截图或技术求助前去除姓名、头像、题目附件 URL token、账号和密钥。
- [ ] 导出的学情只写入受控的 `memory-bank/网站同步/`，由教师确认后再整理，不自动覆盖正式档案。
- [ ] 本地知识同步配置和令牌只保存在被忽略提交的私有文件；同步日志不记录正文和令牌。
- [ ] 每周检查异常登录、批量下载、越权拒绝、模型费用突增和上传失败。

## 导出、撤回与删除

- [ ] 导出前验证请求人身份和授权范围；家长不能借导出获取其他学生或教师私密数据。
- [ ] 导出文件使用受控传输，下载后按约定清理临时副本。
- [ ] 收到更正请求时保留更正记录，不用无痕覆盖破坏教学证据链。
- [ ] 监护人撤回同意后，停止新的 AI 处理和资料同步，说明仍需依法或依约保留的最小记录。
- [ ] 删除前列出数据库记录、Storage 附件、向量片段、导出副本和备份中的影响范围，经教师确认后执行并记录结果。
- [ ] 文件从本地讲义目录消失只会停用知识索引，不等于完成个人信息删除；真正删除必须走单独流程。
- [ ] 删除和恢复能力至少每季度用虚构账号演练一次。

平台删除接口采用两步确认：教师先通过 `account-admin` 的 `request_data_deletion` 建立待处理申请并核对影响范围，再使用申请 ID 和 `DELETE:<学生用户名>` 确认 `delete_student_data`。第二步会删除私有附件并删除 Auth 用户，属于不可轻易恢复的操作；执行前必须另行获得明确同意并确认备份、导出副本和法定留存安排，不能把“申请删除”视为已经授权执行删除。

## 安全事件

- [ ] 发现密钥泄露时先停用相关入口并轮换密钥，检查调用和费用日志。
- [ ] 发现越权或资料误发时立即阻断访问，保存必要审计证据，确认受影响学生、数据和时间范围。
- [ ] 及时通知负责人，并根据实际影响和适用规则决定对学生、监护人、供应商或主管部门的后续通知。
- [ ] 修复后使用虚构账号复测 RLS、签名 URL、AI 检索隔离和周报可见性，再恢复服务。

## 禁止项

- 不把 Service Role Key、AI Key、同步令牌写入前端、Git、讲义或聊天记录。
- 不把公开 GitHub 仓库当作学生文件存储。
- 不用真实学生数据做截图演示、公开 issue、模型调试样例或自动化测试 fixture。
- 不允许 AI 自动发布周报、自动写入正式档案或自动向家长作出评价。
- 不将一次登录成功或一条 RLS 策略视为完整的隐私验收。
