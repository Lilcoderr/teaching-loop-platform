# 中国大陆访问验收表

GitHub Pages、Supabase 和海外模型接口均可能受运营商、地区、DNS 和时段影响。一次电脑端打开成功不能代表学生手机可以稳定使用；上线前和供应商/域名变更后都应重新实测。

## 测试矩阵

每位试点学生至少覆盖：

| 维度 | 最低覆盖 |
|---|---|
| 设备 | 学生实际手机；教师电脑；至少一台家长手机 |
| 浏览器 | 微信内置浏览器（如计划从微信打开）；系统 Safari/Chrome/Edge |
| 网络 | 中国移动蜂窝、中国联通或电信蜂窝、学生家庭 Wi-Fi、教师网络 |
| 时段 | 上课前、晚间作业高峰、周末各一次 |
| 文件 | 1 张照片、多张照片、接近上限的 PDF、中文文件名 |

无法覆盖某家运营商时，在验收记录中明确标为“未测试”，不要写成“通过”。

## 测试步骤

- [ ] 首次打开完整站点，记录是否出现 DNS、TLS、白屏或资源加载失败。
- [ ] 登录、刷新页面、关闭后重开，确认 session 能恢复且不会跳到错误角色。
- [ ] 安装 PWA 或添加到主屏幕，确认图标、启动路径和更新后的版本正常。
- [ ] 上传单图、多图和 PDF；记录文件大小、上传耗时、失败提示与重试结果。
- [ ] 打开私有附件，确认使用授权访问且链接不会暴露其他学生文件。
- [ ] 完成一次学生留言和教师回复。
- [ ] 进行四级 AI 答疑，确认通用回答、讲义引用、错题引用和“未找到已学资料”提示。
- [ ] 在未提交尝试时请求完整解答，确认系统不会返回完整解析。
- [ ] 教师复核一次提交并安排复习，学生侧确认状态变化。
- [ ] 教师发布周报，关联家长可见；未关联家长和其他学生不可见。
- [ ] 关闭 AI 或使用无效测试模型配置，确认提交仍保存并进入人工处理。
- [ ] 断网后重连，确认不会重复创建提交、重复扣费或丢失已填写内容。

## 通过条件与处置

- 核心流程不是以固定秒数一刀切，而是要求学生真实网络下连续三轮完成登录、上传和状态确认，无数据丢失、串号或无提示失败。
- AI 延迟可以降级，但上传与人工队列必须可用；超时应给出明确反馈，不能让学生反复点击造成重复请求。
- 任一目标网络持续无法加载前端、连接 Supabase 或上传附件时，不进入正式试点。先保留人工交作业渠道，再评估更换静态托管、后端区域或国内合规部署。
- 若选择中国大陆境内服务器或自定义域名，应另行核对 ICP 备案、公安备案及供应商的数据合规要求。本仓库不自动完成这些手续。

## 持续监测

- 每周从学生真实网络完成一次小文件上传和一次登录，不只做服务器端探活。
- 记录前端版本、Supabase 状态、函数错误率、Storage 失败率、AI 超时率和当日费用。
- 供应商公告、套餐或域名发生变化后立即复测。
- 平台不应成为唯一交作业入口；始终向学生和家长保留明确的备用提交方式（备用入口必须保留）。
